# UI Guidelines for TODO Application

This document defines visual and accessibility standards for the TODO app interface.

## 1. User material components
- Use Material Design components for consistent UI behavior and accessibility.
- Buttons, dialogs, inputs, cards, and lists should follow Material Design patterns.

## 2. Color palette
- Use a warm, autumn-inspired color palette.
- Primary colors: orange, red, yellow.
- Accent and neutral colors: deep black, charcoal, warm beige.
- Ensure sufficient contrast between text and background for readability.

## 3. Accessibility
- Maintain high accessibility standards (WCAG AA or better).
- Use semantic HTML and proper ARIA landmarks where needed.
- Ensure keyboard navigation works for all interactive components.
- Provide accessible labels for inputs and meaningful alt text for icons.

## 4. Button style
- Use sharp corner buttons (no rounded corners) for primary and secondary actions.
- Keep button sizes comfortable for touch interaction.
- Apply clear hover/focus styles to support usability.
